# 产品方向 (Product Direction)

Aurona Code 旨在成为一款跨平台、高性能、高度现代化的桌面端代码编辑器。在经历了 0.1.0 的技术债扫除与 0.2.0 的 UI 全局重构后，我们已准备好迎接最终形态：**“一个开放的、支持全生态沙盒插件的终极编辑器引擎”**。

## 核心定位

1. **绝对掌控的本地基础设施**
   我们坚持由 Tauri V2 (Rust) 驱动底层核心。不会急于堆砌上层的业务逻辑，而是将构建能够与本地文件系统、命令行 PTY 终端、以及 Git 版本控制无缝交互的极速本地环境作为立身之本。所有的核心视图必须拥有媲美原生的渲染帧率与键盘导航能力。

2. **从 Corona 到 Corona+**
   当前版本已经步入 Corona+ 架构。
   在 0.1.0 时代，我们通过“Corona 架构”实施了前端业务与 Tauri 原生 IPC 的强制隔离；而到了 0.2.0 时代，我们不仅引入了极致的极客玻璃态 (Frosted Glass) 美学，更将隔离推向了极致，为即将到来的 APE（Aurona Plugin Engine）做好了沙盒权限门禁预备。

3. **未来愿景：VSCode 兼容插件生态与 APE 沙盒引擎**
   在基础设施搭建完毕且架构隔离清晰后，我们的长期愿景是：
   打造一套 **APE (Aurona Plugin Engine) 架构系统**，包含两类隔离沙盒：
   - **Aurona 原生沙盒 (WASM/WASI)**：轻量、安全的自定义脚本，能访问有限的 UI 权限。
   - **VSCode 兼容沙盒 (JS/Web Worker)**：旨在直接兼容并运行庞大的 VSCode Extension 生态代码！通过在后台 Web Worker 中模拟 VSCode 的 Node 运行环境与 `vscode` API 注入，让海量的现有扩展（如 ESLint、Prettier、各种语言服务插件）可以在 Aurona Code 中无缝运作。
   
   我们将作为主控者，拦截所有插件的危险调用，保证主程序的绝对安全与性能。通过这条道路，Aurona Code 将直接利用世界级开源生态，突破小型编辑器的局限，成为终极开发利器。
