# 产品方向

Aurona Code 的长期目标是一款现代、高性能、可持续演进的桌面代码编辑器。当前优先级是把已有的编辑、文件、Git、终端、搜索和更新闭环做可靠，而不是用未实现概念扩张产品面。

## 当前阶段

0.3.0 是从功能原型走向可持续开发基线的版本：主要子系统已经可运行，状态、IPC、命令和 Material UI 边界开始稳定，但编辑器异步一致性、完整外部文件监听、跨平台 Release 验证和真实更新链仍未达到成熟产品标准。近期判断应以“是否让现有闭环更可信”为优先，而不是功能数量。

## 已确立的产品原则

1. **保留自研编辑器方向**：AuronaEngine、Rust Rope、Worker 高亮和虚拟视口是当前产品基础，不迁移到 Monaco 或 VS Code 内核。
2. **桌面优先**：文件系统、PTY、Git、窗口和更新是产品核心，Web UI 通过可审计的桌面边界访问它们。
3. **可恢复优先于炫技**：保存、外部变更、revision 冲突、崩溃恢复和资源释放必须先可信。
4. **同一操作只有一条路径**：菜单、快捷键、Fliuno 和按钮共用命令模型。
5. **现代化不等于过度装饰**：界面用材质、层级和密度建立原创性，不把编辑器变成低密度的卡片页面。

## 未来候选（尚未实现）

- 可审查、可撤销的 AI 修改与统一工作区上下文。
- 任务导向工作台和项目架构记忆。
- 具备明确权限、生命周期、版本兼容和错误边界的扩展机制。

插件市场、WASM/WASI 沙箱、VS Code 扩展兼容和 AI Command Center 都没有在 0.3.0 实现。它们只有在核心编辑闭环、权限模型和资源预算稳定后才值得立项。
