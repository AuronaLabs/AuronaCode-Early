# 产品方向 (Product Direction)

Aurona Code 旨在成为一款跨平台、高性能、高度现代化的桌面端代码编辑器。同时，它将作为探索“AI 驱动型本地开发环境”的重要试验场。

## 核心定位

1. **坚实的本地基础设施**
   我们不会急于堆砌上层花哨的功能。我们的首要任务是构建基于 Rust/Tauri 和 React 的、能够与本地文件系统、命令行 PTY 终端和 Git 版本控制无缝交互的极速本地闭环环境。

2. **0.1.0 的奠基：Corona 架构**
   当前 0.1.0 版本的核心战略是**“扫除技术债”**。
   在此之前，前端 UI 组件往往直接越权调用底层 Rust API，导致大量的状态耦合与潜在的进程死锁。通过实施全新的“Corona 架构”，我们将业务层、状态层与 IPC 完全物理隔离。这一层级坚实的防火墙，是为了未来大量复杂的 AI 自动化脚本交互、乃至 Agent 并发调用预留的安全通道。

3. **未来愿景：Agent 原生工作流**
   在基础设施（文件树、Git、LSP、PTY、通知系统）搭建完毕且架构隔离清晰后，我们的长期愿景是：
   探索在编辑器侧边栏或内嵌环境中，引入不仅限于“代码补全”的大语言模型（LLM）能力。该 Agent 将利用底层 `Core` 暴露出的能力直接执行 Git 查看、调用系统命令或检查文件状态，实现真正意义上的深度工作流集成。
