# 设计原则 (Design Principles)

Aurona Code 的设计目标是：**宁静、锋利、现代**。
界面允许使用圆角和灵动的微动效（如全局反馈滑出提示），但绝不能在真实的开发高压环境下显得累赘或刺眼。

## 核心原则 (Core Principles)

- **业务理解优先**：UI 应该是帮助开发者理解项目的工具，而非机械地堆砌各种来自传统 IDE 的面板。
- **圆润与留白**：使用圆角几何形状（如圆角卡片面板），克制对比度，利用内边距和留白来切分高密度的信息块。
- **拒绝无意义的装饰**：不添加任何不能解释状态、优先级或动作的纯装饰性 UI 元素。
- **模块化浮动排版**：避免使用经典的 VSCode 三栏死板布局，默认采用“浮动卡片”悬浮在统一底色之上的设计。
- **安静的颜色**：保持表面元素的层次感但色彩“安静”。仅在深色模式或亮色模式下使用微妙的半透明状态色。
- **克制的动效**：动效（如 Toast 滑出、Tooltip 渐显、折叠面板）仅用于指示状态变更或焦点转移。

## 技术落地规范 (Implementation Rules)

- 采用 **Tailwind CSS** 进行快速布局。
- 采用 **CSS 变量** 作为全局主题令牌（Token），不允许直接将具体的颜色代码写死在组件里。
- 全局只使用 **Tabler React Icons**。
- 所有 UI 组件都应当独立在 `UI/` 设计系统目录中维护，业务直接引入而不重写样式。
