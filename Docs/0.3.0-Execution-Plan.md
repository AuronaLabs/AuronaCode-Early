# Aurona Code 0.3.0 执行记录

本文档以当前仓库为准记录 0.3.0 的边界、所有权、IPC 与验证状态。基线是 0.2.10 提交 `ac6a2ba`。

## 范围

- 保留 Tauri 2、React 19、Zustand、AuronaEngine、Rust Rope、Worker 高亮、LSP、PTY、Git 和搜索。
- 不引入 Monaco、CodeMirror、VS Code 内核、AI Command Center、插件市场、云服务或遥测。
- 保留用户在开始实施前对版本文件与 `public/splash.webp` 的修改。

| 长期状态 | 唯一可写源 | 备注 |
| --- | --- | --- |
| 标签、活动区、底部面板、待关闭项 | `useWorkbenchStore` | AppShell/TitleBar 不保留副本 |

## IPC 与事件流

- 业务代码只通过 `Foundation/Desktop` 和领域 IPC 客户端访问 Tauri。
- `invokeDesktop` 把传输错误转为 `DesktopError { domain, code, message, recoverable, cause }`，传输层不自动 Toast。
- PTY 保留高频事件，但通过类型化监听适配器注册与注销。
- 搜索与性能测试携带 request ID，前端主动取消时后端设置取消标记并清理资源。
- Updater 插件对象在边界转换为 DTO 后才进入 UI。

## 编辑器核心流

```text
input / IME
  -> optimistic projection + operation history
  -> one in-flight batch per path
  -> apply_editor_edits(baseRevision, clientBatchId, edits)
  -> clone Rope, apply sequentially, commit atomically
  -> revision acknowledgement or conflict
```

保存校验 revision 和磁盘指纹，外部变化会被拒绝覆盖。当前未实现全量实时文件监听；这是保存防护，不是完整外部变更 UI。

同一批次中的 UTF-16 坐标按前一项编辑完成后的文本顺序解释。Rust 先在克隆 Rope 上应用整批操作，任一项失败时 Rope、revision 和批次回执均保持不变。revision 冲突时前端保留乐观投影、阻止覆盖保存，并通过恢复快照保全本地内容。

保存请求直接进入每文档操作队列，并捕获发起时的文本 checkpoint。保存完成后只有当前投影仍等于该 checkpoint 才会清除 dirty、恢复快照和待关闭状态；保存期间出现的新输入会继续保持 dirty，并立即刷新最新恢复内容。窗口关闭由 `AppServices` 先阻止默认关闭、刷新全部脏文档恢复快照，再销毁窗口。

0.3.0 的文档打开协议仍传输全量文本，因此完整可编辑文档设有 32 MiB 安全上限。编辑器渲染继续使用虚拟视口和分段高亮，但真正的分页可编辑会话留待后续独立设计。

LSP 文档 URI 统一由 Rust 路径转换命令生成并在前端按路径缓存。Windows 子进程使用 Job Object 管理，停止时先发送 `shutdown`/`exit`，再终止残留进程树并清理读写任务与请求等待者。
