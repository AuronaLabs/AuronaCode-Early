# Aurona Code 0.3.0 技术说明

0.3.0 保留现有技术栈和产品边界，重点收紧架构、编辑持久化、桌面 IPC、命令、性能和质量门禁。

## 已完成

- `useWorkbenchStore` 统一长期工作台状态，`AppServices` 统一启动和注销。
- Tauri 导入收口至 `Foundation/Desktop`，并有静态边界检查。
- 编辑器协议包含 revision、批量原子编辑、client batch 去重、UTF-16 边界和磁盘指纹；同一批次的坐标按前一项编辑完成后的文本顺序解释，任一项失败时整批不提交。
- 保存请求直接进入文档操作队列，并以发起时的前端文本 checkpoint 判定保存后的 dirty 状态；保存期间的新输入不会被误判为已落盘。
- 保存使用临时文件、备份恢复、`sync_all` 和权限保留；权限复制失败会清理临时文件并保留原文件，外部指纹不一致时拒绝覆盖。
- 撤销/重做使用有字节和操作数预算的编辑操作，连续输入 500ms 内合并。
- 脏文档随每次实际内容变化重新 debounce 本地恢复快照，按文档串行写入，并在窗口关闭前强制刷新；编辑器再次打开时可选择恢复或忽略。
- LSP 文件 URI 统一由 Rust 的 `Url::from_file_path` 生成；Windows LSP 进程树由 Job Object 管理，正常停止执行 shutdown/exit，异常退出由 Job Object 兜底回收。
- 完整可编辑文档上限为 32 MiB，超限文件会在创建 Rope 和传输 IPC 快照前以独立只读错误界面拒绝打开。
- Git 阻塞命令进入 `spawn_blocking`，搜索和性能测试支持 request ID 及后端取消。
- 性能数据升级为 schema v3，记录原始样本、截尾均值、P95、CV 和可比较环境键。
- Biome 从 153 个错误和 31 个警告清理到零；新增 Vitest/RTL、PR CI 和 Rust `clippy -D warnings`。
- Aurona Material 已覆盖主题切换、拟物强度、玻璃开关、输入框、菜单、列表、Git 与搜索等关键路径；组件焦点由外层材质表达，不再显示 WebView 默认蓝色内框。
- 深浅主题使用独立拟物档位，浅色模式提供新的低强度 `light`，避免调整浅色参数时意外改变深色表现。

## 未写成已完成的事项

- 未实时监听所有外部文件变化；当前已在保存前通过指纹阻止覆盖。
- LSP 和所有 Worker 结果的 revision 协议尚未全面统一；部分路径使用 request ID 丢弃过期结果。
- 0.3.0 已实现虚拟视口、分段行读取与大文件高亮，但文档打开和前端文本投影仍是全量协议；真正的分页可编辑会话不在本版本临时实现。
- 只挂载活动编辑器以及最多 6 个干净投影的 LRU 尚未实现；在有可验证的文档注册表前不冒险卸载脏会话。
- 0.2.10 和 0.3.0 同机 Release 性能对比尚需通过已打包应用手工采集。
- 插件运行时、AI Command Center、云服务和遥测不属于本版本。
